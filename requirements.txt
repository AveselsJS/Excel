1 WebPack (imports / exports) модули идут по умолчанию.
    1.1 Babel - транскрипт JavaScript кода, который приводит его из новых версий в более старые версии (для браузеров,
которые не поддерживают новый синтаксис).
    1.2 Scss - препроцессор, который мы будем использовать, чтобы не писать на чистом CSS.
    1.3 Eslint - нужен для того, чтобы мы в процессе разработки следили за чистотой кода.
    1.4 2 Mods:
        1.4.1 dev - нужен для того, чтобы разработывать наше приложение.
                Dev Server - нужен для того, чтобы обновлять автоматически код в браузере.
                SourceMaps - исходные карты. Они добавляются для Scss файлов, чтобы понять где мы находимся и какие
                    стили мы написали. будет указывать не на один большой файл, а будет указывать на те модули,
                    где мы вели разработку.
                Eslint - иструмент, который служит дял чистоты кода и дял поддежрания кода в едином стиле.
                Not Minified JS & CSS - во время разработки нам важно понимать как код наш выглядит.
        1.4.2 prod - нужен для того чтобы выпускать куда-либо наше приложение.
                Minified code - необходимо для того, чтобы наш код был сжатым и максимально быстро использовалось.
2 Git - будем работать с ним, поскольку она позволяет следить за версионностью вашего приложения, и чем сложнее код
    и чем сложнее в нём взимодействий, тем он нужнее.

    В большим проектах и больших коммандах всегда используется система Git Flow. Идея вот в чём заключается:
